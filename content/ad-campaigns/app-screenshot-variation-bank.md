## Guardrails

- Keep image text short: one main hook, one support line, one CTA.
- Use `الزبدة` for the brand/app, and `زبدة الأماكن`, `زبدة السفر`, `زبدة العطور` for vertical names.
- Avoid time-sensitive lines unless the scheduled Riyadh time matches the copy. No morning copy outside 05:00-11:59. Avoid `الليلة` unless scheduled from 16:00 onward.
- Before any Buffer action, run `npm run buffer:select -- --date YYYY-MM-DD --current-scheduled N --fill-future`.
- Treat this file as unscheduled source copy. Once a row is used, add it to `content/social_post_register.*`.

## Example Patterns Adapted

The standalone HTML showed eight useful patterns:

1. Identity opener: brand mark plus a real app screen.
2. Product hero: phone/screen as the main object.
3. Score/proof card: one strong number with context.
4. Summary quote: short human-readable verdict.
5. Map/discovery: location browsing as the hero.
6. Praise/complaint split: what people liked vs what to watch.
7. Tabs/features: several app sections shown as a system.
8. Three-angle story: one decision explained from multiple sides.

## Platform Adaptation

### X / Twitter

- Use the wide layout `1600x900`.
- Best rows: proof cards, tabs/features, summary quote.
- Caption should be direct and conversational, usually 2-4 short lines plus URL.

### Instagram Feed

- Use `1080x1350` when possible for more screen space.
- Best rows: score/proof, praise/complaint split, carousel-style explanations.
- Keep the first line emotionally clear, then add one practical benefit.

### TikTok / Reels / Stories

- Use `1080x1920`.
- Best rows: product hero, map/discovery, three-angle story.
- Make the first frame readable in under two seconds. Avoid tiny data tables.
