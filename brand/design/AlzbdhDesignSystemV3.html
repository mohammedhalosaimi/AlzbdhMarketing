Alzbdh — Design System

Complete design system for Alzbdh — an Arabic-first, AI-powered concierge for travel, local lifestyle, offers, and perfumes. This is the only file you need to build new pages or features: tokens, components, voice, RTL rules.

00 How to use this file #

This file is meant to be passed to Claude Code or any developer with the request: "Build a [page] for Alzbdh using this design system." Every section has a preview + ready-to-paste code. No external files needed.

For the developer / Claude Code:
  1. Copy the :root block from Section 01 into your global stylesheet.
  2. Use the React components in Section 06 verbatim — don't rename classes.
  3. Follow layout patterns in Section 09 for every new page.
  4. Wrap your HTML in <div class="zb-app" dir="rtl" lang="ar"> so RTL + Changa apply automatically.
  5. Honour the brand voice in Section 02 — copy is the product.
  6. Arabic content (Voice quotes, Halal labels, Verdict) is preserved verbatim — do not translate it. It's the actual UI string.

02 Brand voice #

Alzbdh speaks like a trusted local friend — warm, direct, opinionated. Saudi/Khaleeji conversational Arabic, never corporate Modern Standard. The product's name («الزبدة» = "the butter / the essence") sets the tone: cut to what actually matters.

Tone pillars

  • Honest: we say "skip this" when it's bad. The Voice quote brand mark literally reads «صراحة» ("honestly").
  • Warm: "حواليك" (around you), not "your area". "يلا" (let's go), not "begin trip".
  • Brief: the zubda. Two-sentence verdicts, not paragraphs.
  • Local: Saudi/Khaleeji idioms («يستاهل», «شفت») — never formal MSA when speaking to the user.
  • Confident, not pushy: we recommend, we don't oversell.

Examples — do / don't

DO ✓
«خصّص ساعتين لتوبكابي على الأقل. الجناح الإسلامي واجب.»
Direct, opinionated, specific.
DON'T ✗
«يُنصح بتخصيص وقت كافٍ لزيارة قصر طوبقابي السلطاني والاستمتاع بمقتنياته القيّمة.»
Formal MSA, vague, sounds like a brochure.

05 RTL & Arabic rules #

RTL is not "mirroring CSS" — it's a layout discipline. These rules are non-negotiable.

1. Use logical CSS properties

margin-inline-start / padding-inline-end / border-inline-start / inset-inline-start — never margin-left or right. Arrows, chevrons, sliders all flip automatically.

2. Mirror directional icons, keep symmetric ones

  • Mirror: back/forward chevrons, arrows, send icon, page turners. Use transform: scaleX(-1) on RTL or pick the correctly-pointing variant from the icon set.
  • Don't mirror: mosque, plane, search, heart, star, clock — symmetric or universal.
  • Numbers in icons: never. Icons are language-agnostic.

3. Embed Latin words correctly

Wrap Latin inline runs (brand names, English-only terms) in <span class="en"> with direction: ltr; unicode-bidi: isolate. This stops the brand "Lattafa" from breaking the Arabic word order around it.

4. Numerals

  • Body text: Eastern Arabic (٠١٢٣٤٥٦٧٨٩) — feels native.
  • Code, URLs, version numbers: Western (0123456789).
  • Prices, times, large numbers: tabular alignment via font-variant-numeric: tabular-nums.

5. Punctuation & quotation

  • Use Arabic comma ، and question mark ؟ in Arabic text — not Latin , or ?.
  • Quotes: French-style guillemets «…» for Voice quotes and editorial nibadhah.

09 Layout patterns #

Every new page follows one of these. Don't invent a new pattern before exhausting these five.

A — List page (mobile)

Hero → filter chip row → vertical stack of cards → sticky CTA on detail-bound items. Use .zb-section-h for "see all" headers between sub-lists.

B — Detail page (mobile)

Image hero (full-bleed, 240–320px) → eyebrow + title → .zb-voice quote (one, max) → meta chips (Halal, rating, distance) → body content → primary CTA fixed-bottom or sticky.

C — Wizard / multi-step (mobile + web)

Step indicator at top (e.g. «٣ من ٧») → ONE question per step → primary CTA fixed-bottom. Don't show two prompts at once. Web version mirrors but adds a live preview pane on the right.

D — Concierge (chat)

Empty state with 3–5 quick-prompt chips ("plan a trip", "what's open now"). User bubbles right (start in RTL), Alzbdh bubbles left, with the green forest avatar. Inline rich cards (place, offer) embed in Alzbdh's bubbles.

E — Web (large screen)

Structure: right sidebar → topbar (search + avatar) → multi-column content. Concierge dock stays floating in lower start.
Rule: don't replicate the mobile experience vertically — exploit the width. Master/detail, split panes, multi-column grids.

10 Accessibility #

A product that serves everyone, including elderly users and people with disabilities.

  • Touch targets: 44×44px minimum for every interactive element.
  • Contrast: body text on background ≥ 4.5:1. Large text ≥ 3:1.
  • Focus state: always visible (outline: 2px solid var(--zb-accent)).
  • Arabic aria-label: every icon-only button needs an Arabic label.
  • Semantic HTML: <button> for buttons (not <div>), <nav> for navigation.
  • Motion: respect prefers-reduced-motion — collapse transitions to opacity only.
  • Screen reader: Eastern Arabic numerals read correctly with lang="ar" + VoiceOver.
  • Language switching: any inline Latin must be wrapped with lang="en" for correct screen-reader pronunciation.

11 Dark mode #

Alzbdh supports full dark mode via [data-theme="dark"] on <html> or .zb-app. Every surface and text color flows from a variable, so themes flip atomically.

Conversion rules:
  1. Surfaces elevate via lightness, not shadow: bg #14110D → card #1F1B16 → elevated #2A251D. No drop shadows on dark — use lighter borders instead.
  2. Green opens up on dark: --zb-accent shifts from #1F3D2E → #7BBF9A for AA contrast. Logo flips to white-on-dark variant.
  3. Text: primary #F5F1E8 (warm cream, never harsh white) · secondary #B8B0A2 · muted #7E776B.
  4. Voice / Verdict / Halal stay semantic: same hues, ~15% lighter (e.g. halal-cert #16864F → #3FAE76).
  5. Photos: apply filter: brightness(.85) to reduce glare.
  6. Borders: rgba(255,255,255,.08) instead of solid lines.

Activation

// 1) Read user preference from localStorage or system
const theme = localStorage.getItem('zb-theme')
            || (matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');

// 2) Apply to <html>
document.documentElement.dataset.theme = theme;

// 3) Toggle (e.g. from settings)
function toggleTheme() {
  const next = document.documentElement.dataset.theme === 'dark' ? 'light' : 'dark';
  document.documentElement.dataset.theme = next;
  localStorage.setItem('zb-theme', next);
}
When to use dark: dark mode is opt-in, not default. Alzbdh's identity is warm cream — light is canonical. Dark is for nighttime comfort and long reading sessions (perfume nibadhah, trip detail). Don't ship features dark-only.

Pick Lucide as the canonical icon library — outline style, 24×24 viewBox, native stroke. Override stroke-width to 1.8. Don't mix line styles (no Heroicons/Material/Tabler in same product). Custom Alzbdh icons (concierge sparkle, halal mark) follow the same metrics.

Sizes

16 chips, inline meta · 20 tab bar, list rows · 24 headers, primary buttons · 32 empty states · 48+ illustrations. Stroke 1.8 at all sizes. Color currentColor — never hardcode.

20 Motion #

Motion should feel warm and confident — never bouncy, never sluggish. Default to short durations and gentle easing.

Tokens

:root {
  /* Durations */
  --zb-d-instant: 100ms;  /* hover, press */
  --zb-d-fast:    180ms;  /* tooltip, chip toggle */
  --zb-d-base:    240ms;  /* dialog open, page tab */
  --zb-d-slow:    360ms;  /* sheet slide, route reveal */
  --zb-d-paced:   560ms;  /* hero reveal, onboarding */

  /* Easings */
  --zb-e-out:     cubic-bezier(.2, .7, .2, 1);     /* default — confident exit */
  --zb-e-in:      cubic-bezier(.6, 0, .8, .3);      /* leaving */
  --zb-e-spring:  cubic-bezier(.5, 1.4, .3, 1);     /* sheet snap (subtle, NOT bouncy) */
}

Patterns

  • Sheet open: transform: translateY(100%) → 0 with --zb-d-slow --zb-e-spring.
  • Page transition: incoming page fades + slides 12px from end edge, --zb-d-base --zb-e-out.
  • Card hover (web): transform: translateY(-2px) + shadow lift, --zb-d-fast --zb-e-out.
  • Voice quote: on first reveal, soft fade-in only — no slide. The quote is content, not chrome.
  • Reduced motion: respect prefers-reduced-motion — collapse all transforms to opacity, halve duration.

24 Breakpoints #

Mobile-first. Each breakpoint adds capability, never overrides intent.

--zb-bp-sm:  480px;   /* large phone */
--zb-bp-md:  768px;   /* tablet portrait */
--zb-bp-lg:  1024px;  /* tablet landscape, small laptop — sidebar appears icon-only */
--zb-bp-xl:  1280px;  /* full desktop — sidebar opens, multi-column unlocks */
--zb-bp-2xl: 1536px;  /* wide — content max-width caps at 1440 */
Layout shifts:
  • 0–767: mobile shell (bottom tab bar, full-bleed sheets, single column).
  • 768–1023: mobile shell with comfortable margins; cards may show in 2 columns.
  • 1024–1279: desktop shell, sidebar collapses to icon-only 64px; main content single-or-double column.
  • 1280+: sidebar opens to 220–260px; pages can use master-detail and 3-column grids.
